Lesson 1: Variables

  • Naming Conventions
    • - Variable names always start with a lowercase letter. Numbers and symbols should NOT be used to start the name of a variable. Spaces are also not allowed in variable names. Variable names should be short, yet descriptive of their purpose. No two variables can have the same name. When writing multiple words in the variable name, there is what is known as snake casing, and camel casing.
    - Snake Casing: Names with snake casing are separated by underscores instead of spaces.
    - Camel Casing: Names with camel casing have no spaces or underscores. Each word in the name starts with a capital letter.
    - Examples:
    int number;
    int oneTwoThree; //camel casing
    char character;
    char characters_and_letters; //snake casing
